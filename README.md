# 小六壬占卜 · v10.5.174 PDF 明细审计误报收口版

## v10.5.174 PDF 明细审计误报收口版

这一版是 PDF 明细模式 Debug 误报的最后一次收口。上一版已经确认 PDF 完整报告、只导出图表、只导出明细都能生成，问题集中在 Debug / 发版体检仍把 CSS 或封面说明里的“图表总览 / chart-section”当成正文图表章节，导致 `analytics-pdf-export-options-v168` 继续失败。

本版不继续改 PDF 生成、不改导出中心、不改历史数据、不改起课和断卦逻辑，只把审计口径收窄到真正应该检查的正文结构。

### 本版新增 / 修复

1. PDF 明细审计误报收口：
   - 新增 `pdfAuditBodyHTML()`：体检前先移除 PDF HTML 里的 `<style>` 和 `<script>`。
   - 新增 `pdfAuditHasChartBody()`：只判断正文是否真的存在 `chart-section`、图表卡片或 `<h2>图表总览</h2>`。
   - 新增 `pdfAuditHasDetailBody()`：判断正文是否存在历史明细、复盘明细或 detail card。
   - 明细模式允许封面 / 导出确认摘要出现“省略图表总览”等说明文字。
   - 明细模式允许 CSS 里保留 `.chart-section` 样式，不再把样式关键词当成正文混入。

2. PDF 三模式体检口径统一：
   - 完整报告：必须有正文图表 + 正文明细。
   - 只导出图表：必须有正文图表，不能有正文明细。
   - 只导出明细：必须有正文明细，不能有正文图表。
   - `pdfModeAudit.full/charts/details` 会显示每种模式的长度、正文图表存在性、正文明细存在性。

3. 发版体检继续绑定真实结构：
   - `analytics-pdf-export-options-v168` 继续检查 PDF 完整 / 图表 / 明细三种模式。
   - 但判断依据改为新结构审计，不再被 CSS 或封面说明误导。
   - 如果这一版后同项仍误报，就建议把 `ANALYTICS_PDF_EXPORT_OPTIONS_MISSING` 按已知误报降级/加入黑名单，不再继续围绕导出功能反复修。

4. 版本记录补齐：
   - 首页版本记录补回 v10.5.173，并新增 v10.5.174。
   - README、manifest、service-worker、页面 title、meta、Debug 版本源同步更新。
