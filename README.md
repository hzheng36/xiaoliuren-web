## v10.5.153 Debug 名单增强与安全修复回滚版

本版基于 v10.5.152 继续按原计划推进：保留 Debug 警告黑名单/白名单、一键安全修复与安全模式基础能力，并进一步增强“名单管理”和“修复可回滚”。

### 本版处理

1. 增强 Debug 警告名单：黑名单/白名单现在支持通配符，例如 `NIGHT_*`、`*_MISSING`，适合批量处理同类误报或复核项。
2. Debug 面板新增“当前非错误加入白名单”，可把当前被黑名单盖住、但想重新观察的问题强制恢复显示。
3. Debug 面板新增“当前问题移出名单”，可把当前问题 code 从黑名单和白名单里一起移出，恢复正常灵敏度判断。
4. Debug 名单状态说明升级：显示默认黑名单数量、通配符支持状态、当前问题匹配结果，方便区分 `rawIssues`、`ignoredIssues`、`blacklistedIssues` 和 `whitelistedIssues`。
5. 一键安全修复新增“恢复修复前设置”：从上一次修复备份中回滚 source 设置，只恢复显示/导航/阅读/Debug 等安全设置，不影响历史、模板和复盘数据。
6. 一键安全修复报告增强：记录移除缓存、备份键数、修复前后清理候选数量、最后修复/回滚时间。
7. 发版体检表升级为“Debug 警告名单增强可用”和“一键安全修复/回滚/安全模式可用”。
8. 同步更新首页版本、版本记录、README、manifest、service-worker 与知识库版本说明，继续保持 1 个主 style + 1 个主 script。

### 这版主要解决什么

这版把 v10.5.152 的名单功能从“能用”升级成“更好管理”：

- 确认误报：放黑名单，后续不再反复提示。
- 需要复查：放白名单，即使黑名单里也会恢复显示。
- 同类误报：可以用通配符统一管理。
- 放错名单：用“当前问题移出名单”快速恢复默认判断。
- 修复后不满意：用“恢复修复前设置”回滚修复前的显示设置。

一键修复、恢复修复前设置、安全模式仍然只处理显示、缓存、本地 source 设置和运行时状态，不会改变小六壬核心起课/占卜逻辑，也不会删除历史、模板或复盘数据。

### 测试建议

1. 上传全部文件后，Safari 强制刷新。
2. 打开 Debug → 一键自检，重点看：
   - `Debug 警告名单增强检查通过`
   - `一键安全修复回滚检查通过`
   - `显示完整性总审计检查通过`
   - `场景重心主线化检查通过`
3. 在 Debug 的“警告名单”里测试通配符：例如临时加入 `NIGHT_*` 到黑名单，再跑一键自检，确认匹配项会进入忽略。
4. 测试“当前非错误加入白名单”：确认白名单会覆盖黑名单。
5. 测试“当前问题移出名单”：确认当前 code 会从黑名单/白名单移出。
6. 谨慎测试“一键修复”后，再点“恢复修复前设置”，确认只回滚显示/导航/阅读/Debug 等 source 设置，不影响历史记录。
