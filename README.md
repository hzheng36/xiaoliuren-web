## 起课规则

本工具的主线规则为三数连续顺推法：

1. 第一数从大安起。
2. 第二数从第一数落宫继续起。
3. 第三数从第二数落宫继续起。

命运场景采用：

> 大安起月，月上起日，日上起时。

并以时宫作为自身宫进行命盘排法。

## 后续可做：日志系统设想

建议后续做「占例日志 / 复盘日志」而不是普通技术日志。它可以和历史记录结合：

- 每条占例增加日志时间线：起课时记录、第一次复盘、第二次复盘、最终结论。
- 复盘状态：待观察、已应验、未应验、部分应验、已作废。
- 日志标签：感情、事业、财运、考试等场景标签外，再增加「重要案例」「可学习」「误判原因」。
- 统计页增加：不同场景的应验比例、常见误判类型、常用起课方式、重点案例数量。
- 导出时把日志一起导出，方便长期保存和复盘。

建议先做轻量版：只给历史记录增加「复盘状态 + 复盘备注 + 重点案例」三项，稳定后再做统计图和筛选。

## GitHub Pages / PWA 缓存提示

如果上传新版后仍显示旧版本，通常是浏览器或 PWA Service Worker 缓存导致。可以尝试：

1. 把 ZIP 解压后的 7 个文件全部上传到仓库根目录，尤其不要漏掉 `service-worker.js`。
2. 在 Safari 里直接打开网页地址，刷新两次。
3. iPhone 可进入「设置 → Safari → 高级 → 网站数据」，删除对应站点数据。
4. 如果是主屏幕图标打开仍旧，先删除主屏幕图标，再重新添加。
