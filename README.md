## v10.5.156 首页时间联动起课增强版

本版基于 v10.5.155 继续推进首页动态时间与时辰起课的联动：在首页“当前时间”模块新增“刷新时间”和“同步到时辰起课”按钮。普通问事可以快速把当前公历、农历和时辰带入时辰起课；命运场景继续支持占问事项留空，并在时间联动时提示用户改为真实出生年月日和出生时辰。

### 本版处理

1. 首页动态时间模块新增“刷新时间”按钮，可手动刷新公历、农历和当前时辰显示。
2. 首页动态时间模块新增“同步到时辰起课”按钮，可自动切换到时辰起课面板，并把当前时间写入起课时间栏。
3. 同步到时辰起课后显示状态提示：普通问事提示检查时间后开始占卜；命运场景提示需要改成真实出生时间。
4. 命运场景继续保持“占问事项可留空”，不强制填写普通问事文本。
5. 新增首页时间联动的暗夜样式覆盖，避免新按钮和状态提示出现白底或低对比。
6. Debug 快照增强 `homeDynamicTime.actions`，检查刷新按钮、同步按钮和状态提示是否存在。
7. 发版体检新增“首页时间刷新/同步到时辰起课可用”。
8. 同步更新首页版本、版本记录、README、manifest、service-worker 与知识库版本说明。

### 这版主要解决什么

这版把 v10.5.155 的“首页动态时间”从单纯显示，升级成可以辅助起课的小工具：

- 普通问事时，看到当前时辰后可以一键带入时辰起课。
- 命运场景下，不会误导用户直接用当前时间当出生时间，而是提醒用户手动改成真实出生时间。
- 新交互继续进入 Debug、暗夜模式和发版体检，避免后续维护时漏检。

### 测试建议

1. 上传全部文件后，Safari 强制刷新。
2. 打开首页，确认“当前时间”模块显示公历、农历、当前时辰。
3. 点击“刷新时间”，确认状态提示更新。
4. 点击“同步到时辰起课”，确认页面跳到起课方式区域，并自动选中“时辰起课”。
5. 普通问事场景下，确认起课时间栏已填入当前时间，点击开始占卜可以正常生成结果。
6. 选择“命运”场景，确认占问事项可留空；点击同步后，状态提示应提醒“改成真实出生时间”。
7. 打开设置 → 界面显示，关闭“显示首页动态时间”，保存后确认时间模块隐藏；再打开确认恢复。
8. 暗夜模式下检查首页动态时间模块、两个按钮和状态提示，不应出现白底、浅色残留或文字低对比。
9. 打开 Debug → 一键自检，重点看：
   - `首页动态时间检查通过`
   - `首页时间联动起课检查通过`
   - `命运场景免填占问事项检查通过`
   - `Debug 修复中心检查通过`
   - `显示完整性总审计检查通过`
