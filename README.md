## 自检使用方式

1. 上传 GitHub 后，用手机或电脑打开页面，建议先强制刷新一次。
2. 点击左下角 `DEBUG`。
3. 点击 `一键自检`。
4. 看顶部结果：`通过` 可以继续测试；`警告` 需要看是否是旧缓存键；`错误` 需要优先修。
5. 如看到旧缓存键警告，可先确认不需要保留旧键，再点 `清理旧缓存`。
6. 点 `复制日志` 或 `保存日志`，可以把 JSON 发回来继续排查。

## 清理旧缓存说明

`清理旧缓存` 只处理旧补丁/旧版本残留键，例如旧阅读字号、旧快捷按钮、旧 Debug、旧 intent/snapshot、旧命运偏好等。

不会删除：

- 历史记录 `xlr_history_v7`
- 最近查看 `xlr_last_view_v7`
- 用户设置 `xlr_user_settings_v9_1`
- 当前源码设置键，例如 `xiaoliuren_ui_settings_source`、`xiaoliuren_reading_mode_source`
- 问题模板、模板收藏、模板使用统计
