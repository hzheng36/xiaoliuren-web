# 小六壬占卜 · v10.5.228 发版复核非阻断收口版

## v10.5.228 发版复核非阻断收口版

这一版专门收口 v10.5.227 Debug 里剩下的 `RELEASE_CHECKLIST_REVIEW`。处理原则是不加入黑名单，而是把它改为“发版人工复核项”：保留可见提示，方便以后发版前核对，但不再当作阻断发布项。

### 本版主要更新

1. 发版复核项非阻断化
   - `RELEASE_CHECKLIST_REVIEW` 已从阻断发布项集合中移出。
   - 不再计入 `blockingIssues / blockerIssues`。
   - 这类项目仍保留在发版体检表里，作为人工核对信息，不加入黑名单。

2. Debug 显示更清楚
   - 发版体检显示从单纯 `130/132 通过` 改为同时显示：通过数、人工复核数、阻断数。
   - 当前场景下应显示为类似：`发版体检：130/132 通过 ｜ 人工复核 2 ｜ 阻断 0`。
   - 这样以后能一眼区分“还有提示”和“真正不能发布”。

3. 修正 v10.5.226 审计误报
   - v10.5.226 首页特性卡片审计原来只认“首页特性卡片排版修复”注释。
   - v10.5.227 已改成“继续保留”，所以这次让审计同时接受新旧注释，避免因为注释文字变化而误报。

4. 修正 v10.5.227 审计误报
   - 普通问事“术语 / 雷达”导航审计只应在结果页导航处于结果模式时检查。
   - 首页或隐藏结果页状态下，不再拿“顶部 / 场景 / 问事 / 起课 / 说明”这些首页导航去校验结果页导航。
   - 这可以解决历史回看关闭后、结果卡片隐藏时仍被普通问事导航审计误判的问题。

5. 新增 v10.5.228 审计
   - 新增 `xlr228ReleaseReviewAudit`。
   - 检查 `RELEASE_CHECKLIST_REVIEW` 是否已经不在阻断发布项集合里。
   - 自检通过时会出现：`v10.5.228 发版复核项非阻断处理检查通过`。

下一版建议继续做小步收口，不要加大功能。优先在 iPhone / iPad Safari 上复测：历史回看、普通问事结果页、命运页、Debug 弹窗、底部导航 active、弹窗底部留白。如果 Debug 已经稳定显示“阻断 0”，就可以进入阶段性封版前的最终可见问题清理版。
