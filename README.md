## v10.5.154 安全模式与修复中心增强版

本版基于 v10.5.153 继续按原计划推进：把一键安全修复、安全模式、回滚备份、修复报告和保护边界集中成 Debug「修复中心」。本版仍然坚持安全边界：只处理显示、缓存、本地 source 设置和运行时状态，不修改小六壬核心起课/落宫/断卦逻辑，也不删除历史、模板或复盘数据。

### 本版处理

1. 新增 Debug「修复中心」面板，集中显示安全模式状态、回滚备份状态、旧缓存候选数量和修复建议。
2. 新增「生成修复报告」：可先生成安全修复建议，不立即改设置，方便判断是否需要一键修复。
3. 新增「刷新修复状态」：不重新跑完整自检，也能快速查看安全模式、备份和旧缓存候选变化。
4. 一键安全修复继续先自动备份，再清理旧补丁缓存、恢复安全阅读/导航/快捷按钮设置并刷新页面状态。
5. 补齐快捷按钮安全设置恢复：一键修复后会恢复主页、历史、分析、重占、电脑、展开、折叠、设置、夜间、置顶、置底等安全按钮显示。
6. 修复中心明确列出保护边界：历史记录、模板、复盘记录、当前占卜结果、JSON 知识库与核心起课逻辑均为保护数据。
7. Debug 快照新增 `repairCenter` 字段，发版体检新增“Debug 修复中心状态/建议/保护边界可用”。
8. 同步更新首页版本、版本记录、README、manifest、service-worker 与知识库版本说明，继续保持 1 个主 style + 1 个主 script。

### 这版主要解决什么

这版不是继续扩大修复范围，而是把修复功能做得更可控、更透明：

- 先看修复建议，再决定是否执行一键修复。
- 修复前自动备份，修复后不满意可以恢复修复前设置。
- 安全模式只是临时恢复稳定显示，不会改变占卜逻辑。
- 黑名单/白名单继续保留，已确认误报可以隐藏，需要复查的项目可以白名单强制显示。
- 修复中心报告会记录建议、保护范围和修复状态，方便以后扩展 Excel/PDF 报告导出。

### 测试建议

1. 上传全部文件后，Safari 强制刷新。
2. 打开 Debug → 一键自检，重点看：
   - `Debug 修复中心检查通过`
   - `一键安全修复回滚检查通过`
   - `Debug 警告名单增强检查通过`
   - `显示完整性总审计检查通过`
   - `场景重心主线化检查通过`
3. 打开 Debug 里的“修复中心”，查看安全模式、回滚备份、旧缓存候选和保护边界是否显示完整。
4. 点击“生成修复报告”，确认只生成报告，不改变历史、模板、复盘和占卜结果。
5. 谨慎测试“一键安全修复”后，再点“恢复修复前设置”，确认只回滚显示/导航/阅读/Debug 等 source 设置。
6. 如 `NIGHT_THEME_COVERAGE_MISSING` 仍在黑名单中，属于已确认的导航按钮视觉审计误报，可保留忽略；需要复查时可加入白名单。
