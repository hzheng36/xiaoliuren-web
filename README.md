## 自检使用方式

1. 上传 GitHub 后，用手机或电脑打开页面，建议先强制刷新一次。
2. 点击左下角 `DEBUG`。
3. 点击 `一键自检`。
4. 看顶部结果：`通过` 可以继续测试；`警告` 需要看是否是旧缓存键；`错误` 需要优先修。
5. 点 `复制日志` 或 `保存日志`，可以把 JSON 发回来继续排查。

## 自检的好处

- 以后不用靠肉眼猜是不是旧版本抢占。
- 可以快速发现重复 ID、导航 target 断掉、折叠状态错乱、旧 Debug 残留、旧版本锁/延迟补丁残留。
- 每次发版前都能做同一套检查，避免“补丁越叠越乱”。
