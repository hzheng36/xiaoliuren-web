## v10.5.155 命运免填与首页动态时间版

本版基于 v10.5.154 继续按原计划推进：优先处理两个后续升级点——命运场景不再需要填写占问事项，首页新增公历/农历/时辰动态时间显示，并可在设置中打开或关闭。Debug 修复中心、安全模式、黑白名单、一键安全修复与回滚能力继续保留。

### 本版处理

1. 命运场景下，“占问事项”改为可留空提示，不再让用户误以为必须填写普通问事文本。
2. 命运场景留空时，历史记录、复制结果和 Debug 中统一显示为“命运命盘 / 未填写占问事项”，避免空白看起来像数据缺失。
3. 命运场景下隐藏普通问事的“问题类型识别”和 A/B 选择面板，减少命盘与普通问事混用。
4. 首页新增“当前时间”模块，动态显示公历时间、农历日期和当前时辰。
5. 设置 → 界面显示中新增“显示首页动态时间”开关；默认开启，关闭后首页更简洁。
6. 新增动态时间暗夜模式样式覆盖，避免新增模块出现白底、浅色文字或低对比问题。
7. Debug 快照新增 `homeDynamicTime` 与 `destinyQuestionMode` 审计字段。
8. 发版体检新增“命运场景免填占问事项可用”和“首页公历/农历/时辰动态时间可用”检查。
9. 同步更新首页版本、版本记录、README、manifest、service-worker 与知识库版本说明，继续保持 1 个主 style + 1 个主 script。

### 这版主要解决什么

这版是体验细节增强版：

- 命运场景更像“命盘起盘”，不再强迫用户套用普通问事逻辑。
- 首页增加时间感，方便用户查看当前公历、农历和时辰。
- 新模块继续接入设置、暗夜模式和 Debug 发版体检，避免功能加上去但自检漏掉。
- 原来的修复中心、安全模式、黑名单/白名单继续保留，不改变核心起课、落宫和断卦逻辑。

### 测试建议

1. 上传全部文件后，Safari 强制刷新。
2. 打开首页，确认“当前时间”模块显示公历、农历、当前时辰。
3. 打开设置 → 界面显示，关闭“显示首页动态时间”，保存后确认首页时间模块消失；再打开确认恢复。
4. 选择“命运”场景，不填写占问事项，直接用时辰起盘生成命盘。
5. 打开历史记录，确认命运空问题显示为“命运命盘 / 未填写占问事项”。
6. 打开 Debug → 一键自检，重点看：
   - `命运场景免填占问事项检查通过`
   - `首页动态时间检查通过`
   - `Debug 修复中心检查通过`
   - `显示完整性总审计检查通过`
   - `场景重心主线化检查通过`
7. 暗夜模式下检查首页动态时间模块，不应出现白底、浅色残留或文字低对比。
