## 这版怎么测试历史记录

1. 上传 7 个文件到 GitHub Pages，并强制刷新一次。
2. 做一次普通问事，点 `保存结果`，再打开 `历史`。
3. 测试历史列表里的 `查看 / 收藏 / 复盘 / 删除`。
4. 在历史窗口测试关键词搜索、场景筛选、日期筛选、只看收藏。
5. 测试 `导出历史 TXT`、`导出历史 JSON`、`导入历史 JSON`。
6. 添加一条复盘，再清空复盘，确认时间线也被清空。
7. 打开 Debug，点 `一键自检`，重点看是否有：`历史记录主命名空间检查通过`、`历史记录 DOM 检查通过`、`历史筛选控件检查通过`、`历史导入控件检查通过`。

## 当前源码结构

- `index.html`：单一主样式、单一主脚本。
- `window.XLRNavCore`：源码内建导航主模块，负责主导航、命盘子导航、导航刷新、跳转与自动居中。
- `window.XLRHistoryCore`：源码内建历史记录主模块，负责历史保存、读取、规范化、收藏、刷新、导入导出支撑。
- `window.XLRDebugCore`：源码内建 Debug 主模块，只读状态、复制/保存日志、一键自检、旧缓存安全清理。
- `initSourceUiModules()`：阅读模式、UI显示、快捷按钮自定义、Debug显示开关。
- `bindDestinySourcePanel()`：命运场景的性别侧重与六亲排法选择。
- `renderDestinyBoard()`：源码主函数直接渲染命盘六亲、地支、六神、五星、婚姻、健康与说明。
