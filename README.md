## v10.5.151 显示完整性总审计与场景重心主线化加强版

本版基于 v10.5.150 继续加强“结果页显示完整性”。上一版已经修复了摘要六格 `undefined` 标签问题；这一版把类似问题提升为总审计能力，并把“本场景判断重心”改为优先读取 JSON 知识库主线，避免新增场景只落回“一般事项”。

### 本版处理

1. 新增 `displayIntegrityAudit`：统一检查结果页空模块、可见但无内容、截断/裁切、`undefined/null/NaN/[object Object]`、导航目标缺失等问题。
2. 新增 `sceneFocusAudit`：检查“本场景判断重心”是否有 4 个卡片、是否出现坏文本、是否错误泛化为“一般事项”。
3. 重写 `buildSceneFocusHTML` 主逻辑：优先读取 `XLRKnowledgeCore.getScenePack()` 的场景标题、说明、focus 和 advice。
4. 对没有 JSON 专项库的场景保留安全兜底，但 Debug 会标明来源，方便后续补库。
5. 发版体检表新增“显示完整性总审计通过”和“本场景判断重心主线化可用”。
6. 继续保留 v10.5.150 的摘要标签完整性检查，并把它纳入总审计。
7. 同步更新首页版本、版本记录、README、manifest、service-worker 与知识库版本说明，继续保持 1 个主 style + 1 个主 script。

### 这版主要解决什么

这版不是只修某一个截图，而是把“内容显示不完整”的同类问题集中纳入 Debug：以后如果有模块空白、文字变成 `undefined`、导航指向不存在、内容被 overflow 裁切、或者场景重心没有接入新场景，都可以从一键自检里看到具体报告。

### 测试建议

1. 上传全部文件后，Safari 强制刷新。
2. 从历史记录打开“中奖”“合同”“父母”“宠物”“车辆/交通”等扩展场景。
3. 检查“本场景判断重心”是否不再泛化为“一般事项”。
4. 打开 Debug → 一键自检，重点看：
   - `显示完整性总审计检查通过`
   - `场景重心主线化检查通过`
   - `结果摘要标签完整性检查通过`
5. 之前 `jump-chip` 暗夜视觉审计提示仍属于已知非阻塞项，可以继续按标准模式降级处理。
