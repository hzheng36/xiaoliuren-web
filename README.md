## 自检使用方式

1. 上传 GitHub 后，用手机或电脑打开页面，建议先强制刷新一次。
2. 点击左下角 `DEBUG`。
3. 点击 `一键自检`。
4. 看顶部结果：`通过` 可以继续测试；`警告` 需要看是否是旧缓存键；`错误` 需要优先修。
5. 如看到旧缓存键警告，可先确认不需要保留旧键，再点 `清理旧缓存`。
6. 点 `复制日志` 或 `保存日志`，可以把 JSON 发回来继续排查。

## 清理旧缓存说明

`清理旧缓存` 只处理旧补丁/旧版本残留键，例如旧阅读字号、旧快捷按钮、旧 Debug、旧 intent/snapshot、旧命运偏好等。

不会删除：

- 历史记录 `xlr_history_v7`
- 最近查看 `xlr_last_view_v7`
- 用户设置 `xlr_user_settings_v9_1`
- 当前源码设置键，例如 `xiaoliuren_ui_settings_source`、`xiaoliuren_reading_mode_source`
- 问题模板、模板收藏、模板使用统计

## 当前源码结构

- `index.html`：单一主样式、单一主脚本。
- `window.XLRNavCore`：源码内建导航主模块，负责主导航、命盘子导航、导航刷新和跳转。
- `window.XLRDebugCore`：源码内建 Debug 主模块，只读状态、复制/保存日志、一键自检、旧缓存安全清理。
- `initSourceUiModules()`：阅读模式、UI显示、快捷按钮自定义、Debug显示开关。
- `bindDestinySourcePanel()`：命运场景的性别侧重与六亲排法选择。
- `renderDestinyBoard()`：源码主函数直接渲染命盘六亲、地支、六神、五星、婚姻、健康与说明。
