# 小六壬占卜 · v10.5.196 历史中心总整理与性能轻量化版

## v10.5.196 历史中心总整理与性能轻量化版

这一版承接 v10.5.195 的“历史中心移动端体验与按钮事件总检版”，继续做集中收口。重点不是继续堆新功能，而是降低历史记录中心在打开、筛选、切换视图和移动端长列表浏览时的渲染压力，让前面几版新增的经验库、复盘、导出和章节统计更轻量、更稳定。

### 本版新增 / 修复

1. 历史中心性能轻量化面板
   - 新增“历史中心总整理与性能轻量化”面板；
   - 显示当前面板总数、可见面板、延迟刷新面板、列表渲染数量和本次渲染耗时；
   - 明确提示轻量化只影响 DOM 渲染和面板刷新时机，不修改历史、复盘、导出和核心占卜逻辑。

2. 统计面板按当前视图轻量刷新
   - 历史中心现在先应用面板整理状态，再刷新统计面板；
   - 精简视图、复盘视图、导出视图只即时刷新当前可见面板；
   - 隐藏或折叠的高级统计面板会标记为延迟刷新，切换视图或展开后再刷新，减少首次打开压力。

3. 长列表轻量渲染
   - 历史记录列表在记录很多时默认只渲染前 120 条；
   - 搜索、统计、导出、批量范围仍按当前全部筛选结果计算，不会丢数据；
   - 新增“本次显示全部记录”和“恢复轻量显示”按钮，方便需要完整浏览时手动展开。

4. 移动端长列表流畅度增强
   - 减少移动端一次性插入过多历史卡片导致的卡顿；
   - 时间线视图和列表视图都使用同一套轻量显示策略；
   - 继续保留 iPad / Safari 的弹窗滚动和 safe-area 兼容。

5. Debug / 发版体检增强
   - 新增 `history-performance-light-v196` 发版体检项；
   - Debug 历史模块快照新增 `performanceAudit`；
   - 新增性能面板暗夜/阅读模式覆盖检查，继续防止新增内容出现浅灰底、白块、浅紫字和低对比文字。

6. 版本源同步
   - 页面 title、meta、首页版本标题、版本记录、README、manifest、service-worker 已同步到 v10.5.196；
   - 主线 source 已统一到 `mainline-v10.5.196`；
   - service worker cache 已更新为 `xiaoliuren-v10.5.196`。

### 本版检查

- 已检查主脚本语法。
- 已检查 manifest JSON。
- 已检查旧版本 source 残留。
