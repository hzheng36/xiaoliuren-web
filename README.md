## v10.5.159 历史记录导出增强版

本版基于 v10.5.158 继续升级“导出/报表层”。上一版已经把 Debug 报告导出做成 JSON、TXT 和排查摘要；这一版先不碰核心占卜逻辑，也不急着做真正 Excel/PDF 文件，而是先把历史记录和复盘记录整理成 Excel 友好的 CSV 结构化数据，为后续 Excel 报表、PDF 图表报告打基础。

### 本版处理

1. 历史记录弹窗新增“导出历史 CSV”。
2. 历史记录弹窗新增“导出复盘 CSV”。
3. 历史分析弹窗的“分析导出”里也新增历史 CSV 与复盘 CSV 入口。
4. 历史 CSV 按“一条占卜一行”输出，包含时间、场景、识别类型、起课方式、三数、落宫、综合判断、直接答案、复盘条数、最新复盘状态等字段。
5. 复盘 CSV 按“一条复盘一行”输出，包含关联历史记录、复盘序号、准确度、准确单位、实际结果、备注、复盘时间与更新时间等字段。
6. CSV 自动加入 UTF-8 BOM，减少 Excel / Numbers 打开时中文乱码的概率。
7. CSV 单元格加入公式注入保护：如果内容以 `= + - @` 开头，会自动加单引号，避免被表格软件当公式执行。
8. Debug 快照新增 `historyExport` 字段，用来检查历史 CSV 与复盘 CSV 导出中心是否完整挂载。
9. 一键自检新增 `HISTORY_EXPORT_CSV_MISSING` 检查。
10. 发版体检新增“历史记录 CSV / 复盘 CSV 导出可用”。
11. 同步更新首页版本、版本记录、README、manifest、service-worker 与知识库版本说明。

### 这版主要解决什么

TXT 适合阅读，JSON 适合备份恢复，但两者都不适合直接做报表。CSV 更适合后续进入 Excel、Numbers、Power BI 或 Python 分析，所以这一版先把数据结构打平，避免以后直接做 Excel/PDF 时又要重写历史数据整理逻辑。

### 测试建议

1. 上传全部文件后，Safari 强制刷新。
2. 打开历史记录，确认“历史记录”工具组里出现：导出历史 TXT、导出历史 CSV、导出复盘 CSV、导出历史 JSON。
3. 点击“导出历史 CSV”，确认能下载 `.csv` 文件，打开后中文正常显示。
4. 如果有复盘记录，点击“导出复盘 CSV”，确认一条复盘对应一行。
5. 打开历史分析 → 分析导出，确认也能看到历史 CSV 与复盘 CSV。
6. 打开 Debug → 一键自检，确认没有错误或警告。
7. 在 Debug JSON 中查看 `historyExport.ok`，应为 `true`。
8. 一键自检通过项中应出现“历史 CSV / 复盘 CSV 导出检查通过”。
9. 回归 v10.5.158 的测试点：Debug 导出 JSON、导出 TXT、复制排查摘要仍可用。
