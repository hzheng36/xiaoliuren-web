## v10.5.150 专项摘要标签完整修正版

本版基于 v10.5.149 修复一个真实显示问题：部分 JSON 专项场景在结果页“摘要六格”中，标签会显示成 `undefined`，造成结果看起来不完整。典型场景包括中奖、合同、投资、官司、朋友、子女、父母、宠物、车辆/交通等由 JSON 知识库扩展出来的场景。

### 本版处理

1. 修复结果摘要六格标签不足时显示 `undefined` 的问题。
2. 新增 `normalizeQuickLabels` 兜底逻辑：任何场景的摘要标签不足 6 个时自动补齐。
3. 不再把 JSON 知识库字段名直接错位当成摘要指标标签，避免“中奖概率”对应“中上”这种不够准确的显示。
4. 为中奖等专项场景补充更匹配的摘要标签，例如：中奖倾向、兑票节奏、冲动风险、结果落点、助力情况、娱乐气象。
5. Debug 新增“结果摘要标签完整性检查”，以后若摘要区出现空标签、`undefined`、`null` 或空值，自检会直接报出。
6. 同步更新首页版本、版本记录、README、manifest、service-worker 与知识库版本说明，继续保持 1 个主 style + 1 个主 script。

### 这版主要解决什么

上一版 Debug 日志整体是 pass，但截图中结果页摘要卡片出现了多个 `undefined`。这不是 iOS Safari 误报，而是 JSON 专项场景标签数组不足 6 个导致的真实显示问题。

v10.5.150 已把它收进主源码，不使用后置补丁。

2. 从历史记录里打开刚才那条“中奖”记录。
3. 拉到结果摘要六格位置，确认不再出现 `undefined`。
4. 打开 Debug → 一键自检，重点看是否出现“结果摘要标签完整性检查通过”。
5. 之前那个 `jump-chip` 暗夜视觉审计提示仍属于已知非阻塞项，可以继续按标准模式降级处理。
