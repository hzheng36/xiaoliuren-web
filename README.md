## v10.5.152 一键修复与 Debug 警告名单基础版

本版基于 v10.5.151 继续按原计划推进：在保留“显示完整性总审计”和“场景重心主线化”的基础上，新增一键安全修复、安全模式基础能力，并为类似 `NIGHT_THEME_COVERAGE_MISSING` 的已确认误报加入 Debug 警告黑名单/白名单机制。

### 本版处理

1. 新增 Debug 警告黑名单/白名单：黑名单中的 warning 会进入 `ignoredIssues`，不再反复提示；白名单优先级高于黑名单，可强制恢复显示。
2. 默认把已确认的 `NIGHT_THEME_COVERAGE_MISSING` 导航按钮视觉审计偏差放入黑名单，但仍保留在 `rawIssues` 里备查。
3. Debug 面板新增“警告名单”管理区，可保存名单、恢复默认、把当前非错误问题加入黑名单。
4. 新增一键安全修复基础版：修复前自动备份本地 source 设置，清理旧补丁缓存，恢复安全阅读/导航/快捷按钮设置，并刷新导航与结果页显示。
5. 新增安全模式基础开关：只调整显示、缓存和本地设置，不改变小六壬核心起课/占卜逻辑，不删除历史、模板和复盘数据。
6. 发版体检表新增“Debug 警告白名单/黑名单可用”和“一键安全修复/安全模式基础能力可用”。
7. 同步更新首页版本、版本记录、README、manifest、service-worker 与知识库版本说明，继续保持 1 个主 style + 1 个主 script。

### 这版主要解决什么

这版把“已确认误报”和“需要继续观察的问题”分开管理：

- 黑名单：确认不是问题、暂时不想再被提醒的 warning。
- 白名单：即使它在黑名单里，也要强制显示并继续观察的 warning。
- `rawIssues`：保留原始诊断，方便之后回看，不会真正丢失信息。

同时，一键修复和安全模式只处理显示、缓存、设置和运行时状态，不会偷偷改核心占卜逻辑。

### 测试建议

1. 上传全部文件后，Safari 强制刷新。
2. 打开 Debug → 一键自检，重点看：
   - `Debug 警告名单检查通过`
   - `一键安全修复基础检查通过`
   - `显示完整性总审计检查通过`
   - `场景重心主线化检查通过`
3. 检查 `NIGHT_THEME_COVERAGE_MISSING` 是否进入 `ignoredIssues` / 黑名单忽略，而不是继续作为 warning 干扰。
4. 在 Debug 的“警告名单”里测试：把 `NIGHT_THEME_COVERAGE_MISSING` 加入白名单后，再跑一键自检，确认它会恢复显示。
5. 谨慎测试“一键修复”和“安全模式”：它们会备份并重置显示设置，但不会删除历史、模板、复盘数据。
