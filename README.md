# 小六壬占卜 · v10.5.195 历史中心移动端体验与按钮事件总检版

## v10.5.195 历史中心移动端体验与按钮事件总检版

这一版承接 v10.5.194 的“经验库总修复与面板整理版”，继续按“集中修复”路线推进。重点不是继续堆新统计面板，而是把历史记录中心在手机、iPad 和 Safari 下的使用体验再整理一轮：弹窗高度、滚动可达性、按钮点击区域、批量操作提示、按钮事件挂载，以及新增内容的暗夜 / 阅读模式体检。

### 本版新增 / 修复

1. 历史中心移动端体验总检
   - 新增“历史中心移动端体验与按钮事件总检”面板；
   - 集中检查 iPad / Safari 弹窗高度、整窗滚动、底部安全留白和历史列表可达性；
   - 体检只检查显示与交互，不修改历史记录、复盘数据、起课规则或断卦逻辑。

2. 移动端 / iPad / Safari 弹窗滚动增强
   - 历史弹窗在移动端改用更稳定的 `100dvh` 与 safe-area 留白；
   - 移动端下历史中心整体可滚动，避免上方筛选器和统计面板过多时挤压列表；
   - iPad / Safari 下继续启用 `-webkit-overflow-scrolling`，让长内容滚动更顺。

3. 按钮点击区域总整理
   - 历史模式按钮、案例筛选按钮、面板整理按钮、复盘/导出工具栏、批量管理按钮统一增强触控高度；
   - 手机端工具栏自动改成双列或单列，减少按钮过窄、误触和挤压；
   - 保留桌面端原有布局，不影响电脑端使用。

4. 批量操作提示增强
   - 批量管理状态继续显示已选数量、当前显示数量和撤销状态；
   - 批量收藏、取消收藏、高价值、标签、归档、删除、选中 JSON/CSV 导出和撤销入口纳入按钮事件总检；
   - 继续保留删除前撤销备份机制。

5. Debug / 发版体检增强
   - 新增 `history-mobile-event-audit-v195` 发版体检项；
   - Debug 历史模块快照新增 `mobileEventAudit`；
   - 新增移动端触控、按钮事件、批量提示、暗夜/阅读模式继承检查；
   - 后续新增按钮、弹窗、统计面板和导出入口，也必须同步进入移动端与暗夜体检。

6. 版本源同步
   - 页面 title、meta、首页版本标题、版本记录、README、manifest、service-worker 已同步到 v10.5.195；
   - 主线 source 已统一到 `mainline-v10.5.195`；
   - service worker cache 已更新为 `xiaoliuren-v10.5.195`。

- 已检查 manifest JSON。
- 已检查旧版本 source 残留。
- 已检查 PDF 旧版本硬编码残留。
