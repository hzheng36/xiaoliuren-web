2. 底部导航模式和顶部导航模式同步调整，避免只修一个方向导致体验不一致。

### v10.5.116 导航起点与高亮跟随修正版

本版基于 v10.5.115 继续修复自适应导航。你这次截图里的问题是：导航项较多、宽度受限时，上一版为了让 active 靠中，会自动把导航条横向滚到中间，导致一级/二级导航看起来不是从第一个按钮开始。

1. 一级导航和命盘二级导航改成“起点优先”：进入结果页、进入命盘区域、刷新或重新渲染时，默认先从第一个按钮开始显示。
2. 横向跟随逻辑从“强制居中”改为“出界轻微跟随”：当前 active 按钮仍在可视范围内时，导航条不再自动移动；只有 active 快出界时，才轻微滚动到可见范围。
3. 绿色高亮 `slot-active` 不再按导航中线选按钮，而是始终跟随真实 active 按钮，避免出现“结果 active，但答案变绿”这类错位。
4. 继续保留 v10.5.115 的自适应宽度：按钮少时底板变短，按钮多时仍然横向滑动。
5. Debug 导航状态升级到 `beta-v10.5.116`，可以继续观察 `scrollLeft`、`overflow`、`currentTarget`、`active`、`slotActive` 是否一致。
6. 同步更新首页版本、版本内容更新、README、manifest 和 service-worker 缓存名，避免旧版本跳动。
7. 继续保持 7 个完整文件交付，无后置补丁、无延迟抢写补丁。
