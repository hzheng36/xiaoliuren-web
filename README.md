## 起课规则

本工具的三数、时辰、事件起课采用 **三数连续顺推法**：

1. 第一数从大安起数。
2. 第二数从第一数落宫处继续顺推。
3. 第三数从第二数落宫处继续顺推。

六宫顺序：

```text
大安 → 留连 → 速喜 → 赤口 → 小吉 → 空亡
```

### 起课注意事项

- 问题要清晰、具体。
- 一事不二占，一时不起二课。
- 问事时心念专一，不疑不妄。

### 月日时起课

月日时起课遵循：**月上起日，日上起时**。

示例：农历一月初九，午时。

1. 从“大安”起一月，一月落大安。
2. 从大安起初一，顺数至初九，落速喜。
3. 从速喜起子时，顺数至午时（第 7 位），仍落速喜。
4. 最终落位：速喜。

时辰起课以用户设备显示的当地时间为准；数字起课与事件起课不依赖地理位置。

### 单数起课

单数起课只用一个数字，从大安起数得到一个主宫，适合快速判断吉凶。当前网页中会以该主宫贯穿三象显示，方便使用同一套结果页面、历史记录和复盘功能。

## 本地数据说明

历史记录、复盘、模板、阅读设置等均保存在当前浏览器本地。换设备或清理缓存前，建议先导出 **完整备份 JSON**。

## 导入导出

- TXT：用于阅读、分享、给后续分析参考。
- 历史 JSON：只迁移历史记录、收藏、复盘和 A/B 数据。
- 模板 JSON：只迁移问题模板。
- 完整备份 JSON：迁移整个个人占卜台，包括历史、复盘、模板和设置。

## PWA 离线缓存说明

本版新增 `service-worker.js`。首次通过 GitHub Pages 正常访问后，浏览器会缓存核心文件。之后在网络较差或离线时，已添加到主屏幕的工具仍可打开已缓存版本。

如果上传新版本后手机仍显示旧内容，可以尝试：

1. 在 Safari 中刷新网页。
2. 关闭后重新打开主屏幕图标。
3. 必要时在 iPhone 设置中清理 Safari 网站数据后重新添加。

## 部署到 GitHub Pages

将以下文件放到仓库根目录：

```text
index.html
README.md
manifest.webmanifest
service-worker.js
apple-touch-icon.png
icon-192.png
icon-512.png
```

然后在 GitHub 仓库 Settings → Pages 中选择 main / root 发布。

## iPhone / iPad 添加到主屏幕

1. 用 Safari 打开 GitHub Pages 网址。
2. 点击分享按钮。
3. 选择「添加到主屏幕」。
4. 图标会使用 `apple-touch-icon.png`。
